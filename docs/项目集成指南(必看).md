### 第 1.5 步：各开发人员启用 Git 提交规范

**克隆项目后第一件事，运行以下命令：**

```bash
git config core.hooksPath githooks/
```

设置后 commit message 必须符合 `<type>: <描述>` 格式，否则会被拒绝提交。这是项目统一的提交规范，每人必须设置。

### 第 2 步：各模块并行开发

### 第 3 步：PR 合入 dev 分支

各模块完成开发后，向 `dev` 分支发起 Pull Request。

### 第 4 步：项目负责人 集成测试

1. 拉取所有模块的变更到 `dev`
2. 验证依赖注入链
3. 执行端到端测试

### 第 5 步：依赖注入（已配置）

`app.py` 的 `create_app()` 中已注入以下依赖链（使用 SQLiteRepository）：

```python
from repositories import SQLiteRepository
from services import AuthService, DataService, CleanService, VisualizeService, AnalyzeService

repo = SQLiteRepository(app.config["DATABASE_PATH"])
app.auth_service = AuthService(repo)
app.data_service = DataService(repo)
app.clean_service = CleanService(repo)
app.visualize_service = VisualizeService(repo)
app.analyze_service = AnalyzeService(repo)
```

> **注意**：当前使用 `SQLiteRepository` 而非 `FileRepository`。数据库文件路径由 `config.py` 中的 `DATABASE_PATH` 控制，默认 `analysis.db`。

## 五、常见集成问题

### 5.1 Service 获取不到

**问题**: Route 中 `current_app.xxx_service` 抛出 `AttributeError`

**原因**: `app.py` 中未将 Service 实例挂载到 `app` 上

**解决**: 确保 `create_app()` 中有类似:

```python
app.data_service = DataService(repo)
```

### 5.2 JS 函数未定义

**问题**: 浏览器 Console 报 `Uncaught ReferenceError: xxx is not defined`

**原因**: HTML 中 JS 文件加载顺序有误

**解决**: 各模块 JS 必须在 `app.js` 之前加载，确保 `app.js` 能调用到模块暴露的函数。

### 5.3 Flask 找不到蓝图

**问题**: `Registering blueprint` 时提示模块不存在

**原因**: Route 模块抛出了 `NotImplementedError`（被 import 时触发）

**解决**: 暂用空函数体替换 `raise NotImplementedError`，或由 项目负责人 先注册空蓝图。

### 5.4 跨域问题（开发调试）

如果前端单独启动开发服务器，需要在 Flask 端启用 CORS：

```python
from flask_cors import CORS

def create_app():
    app = Flask(__name__)
    CORS(app)  # 全局启用
    ...
```

## 六、各模块开发人员注意事项

### 数据管理模块
- 上传后返回的 `dataset_id` 会作为后续所有操作的标识，确保其正确性
- `export` 路由返回的是文件流，不是 JSON，注意 Content-Type 和 Content-Disposition

### 数据清洗模块
- 清洗产生**新数据集**，返回**新的** `dataset_id`，前端会用新 ID 更新全局状态
- `_handle_missing` 和 `_detect_outliers_iqr` 是内部方法，不需要在 Route 中直接调用

### 可视化模块
- 选择 Matplotlib 或 Plotly 后，需要通知 Web 界面模块，因为：
  - Matplotlib: 前端只需 `<img>` 显示 base64
  - Plotly: 前端需引入 Plotly.js CDN + 调用 `Plotly.newPlot()`

### 分析功能模块
- scikit-learn 返回 numpy 类型，必须转为 Python 原生类型（`.tolist()`, `float()`）

### Web 界面模块
- 负责维护全局 `currentDatasetId` 状态，确保每次请求都携带正确的 ID
- 负责协调各模块 JS 的集成顺序和初始化时机
