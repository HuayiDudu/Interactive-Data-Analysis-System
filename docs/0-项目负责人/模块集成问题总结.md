# 模块集成问题总结

---

### 一、依赖与运行环境

1. **pyarrow 版本锁死在 Python 3.14 不兼容的版本** — `==18.*` 在 Python 3.14 上无 wheel，pip 尝试源码编译失败。根源是版本约束写死且未考虑 Python 版本兼容性。

2. **所有依赖用 `==x.*` 紧锁** — 换到不同 Python 版本就可能装不上。应改为 `>=min` 最小版本约束，让 pip 根据当前环境选可用 wheel。

---

### 二、数据流中的类型安全隐患

3. **NaN → JSON 序列化断裂** — `clean_service.py` 直接 `.values.tolist()` 输出 `float('nan')`，Flask `jsonify` 序列化成非法 JSON `NaN`，前端 `response.json()` 崩溃。同是预览数据，`data_service.py` 做了 NaN→None 转换，但 `clean_service.py` 没做，**两个 Service 实现不一致**。

4. **重复列名未处理** — CSV 中 `"年龄","年龄"` 两列同名，pandas 保留。后续 `sort_values`/`groupby` 因列名不唯一报错 400。应在数据解析层统一去重。

---

### 三、模块间接口不一致

5. **`handlePlot()` 无返回值，但调用方期待有** — `plot.js` 的 `handlePlot` 内部渲染完图表就结束了，没 `return data`。`app.js` 拿到的 `result` 是 `undefined`，直接读 `result.plotly_json` 崩溃。**模块暴露的接口与调用方的约定不一致**。

6. **`#plotly-chart` DOM 元素的隐式约定** — `plot.js` 用 `container.innerHTML` 覆盖内容时销毁了 `#plotly-chart`，但 `app.js` 仍通过 ID 引用它做截图。**两个模块通过 DOM 元素 ID 隐式耦合**，一方改了 DOM 结构另一方就崩。

---

### 四、前端状态管理

7. **切换图表类型重置 XY 选中值** — `populatePlotColumns` 不分场景一律清空重建下拉框，用户选的列丢失。手动切 XY 不会触发重建，但切图表类型会，**行为不一致**。

8. **失败重试路径未覆盖** — 请求失败后容器显示错误信息（覆盖了图表元素），用户修正参数再点生成时，`getElementById('plotly-chart')` 返回 `null`，直接崩溃。**错误处理只考虑了首次展示，没考虑后续重试**。
