# 分析功能模块 — 前端接入说明

**交付对象**：Web 界面模块开发人员
**说明**：分析功能模块后端已实现 6 种算法，前端需要做对应修改才能完整使用。本文档说明后端提供了哪些接口和参数，以及前端需要满足的行为。

---

## 一、后端提供的 HTTP API

### 1.1 统一分发路由（推荐）

```
POST /analyze
Content-Type: application/json

{
    "dataset_id": "<string>",
    "algorithm": "<string>",
    ...算法自定义参数
}
```

`algorithm` 可选值：

| algorithm | 说明 | 额外参数字段 |
|-----------|------|-------------|
| `kmeans` | K-Means 聚类 | `n_clusters`（int，2-10，默认 3） |
| `dbscan` | DBSCAN 密度聚类 | `eps`（float，默认 0.5）、`min_samples`（int，默认 5） |
| `linear_regression` | 线性回归 | `feature_cols`（string[]）、`target_col`（string） |
| `polynomial_regression` | 多项式回归 | `feature_col`（string）、`target_col`（string）、`degree`（int，2-5，默认 2） |
| `compare_clustering` | 聚类对比（K-Means vs DBSCAN） | `n_clusters`、`eps`、`min_samples` |
| `compare_regression` | 回归对比（线性/Ridge/Lasso/多项式） | `feature_cols`、`target_col`、`degree` |

响应格式与项目规范一致：
```json
// 成功
{ "status": "success", "data": { ... } }
// 失败
{ "status": "error", "message": "具体错误原因" }
```

### 1.2 按算法独立路由（供灵活调用）

| 路由 | 方法 | 参数 |
|------|------|------|
| `POST /analyze/kmeans` | K-Means | `dataset_id`, `columns`(可选), `n_clusters` |
| `POST /analyze/dbscan` | DBSCAN | `dataset_id`, `columns`(可选), `eps`, `min_samples` |
| `POST /analyze/regression` | 线性回归 | `dataset_id`, `feature_cols`, `target_col` |
| `POST /analyze/poly_regression` | 多项式回归 | `dataset_id`, `feature_col`, `target_col`, `degree` |
| `POST /analyze/compare/clustering` | 聚类对比 | `dataset_id`, `columns`, `n_clusters`, `eps`, `min_samples` |
| `POST /analyze/compare/regression` | 回归对比 | `dataset_id`, `feature_cols`, `target_col`, `degree` |

## 二、前端需要新增/修改的内容

### 2.1 算法选择下拉框

前端需在 `#algorithm-type` 下拉框中增加 4 个选项：DBSCAN 密度聚类、多项式回归、聚类对比、回归对比。

切换算法时，`populateAlgorithmParams(algorithm, columns)` 会被调用，前端需根据 algorithm 值显示对应的参数配置面板。

### 2.2 新增参数配置面板

前端需要为第 1 节表格中的算法新增参数输入控件：

| 算法 | 需要的参数控件 |
|------|---------------|
| DBSCAN | `eps` 数值输入、`min_samples` 数值输入 |
| 多项式回归 | 特征列下拉框（单选）、目标列下拉框、阶数数值输入（2-5） |
| 聚类对比 | K 值调节（可复用 K-Means 的控件）+ DBSCAN 的 eps/min_samples |
| 回归对比 | 特征列下拉框（多选）+ 目标列下拉框（可复用线性回归的控件）+ 阶数数值输入 |

### 2.3 handleAnalyze(datasetId) 的参数组装

当前 `handleAnalyze` 只组装了 kmeans 和 linear_regression 的参数。需改为根据 `algorithm-type` 的当前值读取对应的参数控件值，组装成第 1.1 节的参数字典，然后调用 `POST /analyze`。

### 2.4 渲染函数映射

当前 `handleAnalyze` 内部的 renderMap 只有 kmeans 和 linear_regression 两项。后端新增的 4 种算法各自有对应的渲染函数（已在 `analyze.js` 中实现），需在 renderMap 中补充映射：

| algorithm | 渲染函数 |
|-----------|---------|
| `dbscan` | `_renderDbscanResult` |
| `polynomial_regression` | `_renderPolyRegressionResult` |
| `compare_clustering` | `_renderCompareClustering` |
| `compare_regression` | `_renderCompareRegression` |

### 2.5 历史记录标签

`app.js` 中 `btn-analyze` 的 click 处理目前只识别了 `kmeans` 和 `linear_regression`，需补充其余 4 种算法的显示标签（算法名称 + 关键参数值）。

---

## 三、后端数据校验说明

后端会在 Service 层对以下情况返回 422 错误（前端应做相应提示）：

- 选中的列包含非数值类型（线性回归/多项式回归）
- K 值不在 2-10 范围内
- eps ≤ 0 或 min_samples < 1
- degree 不在 2-5 范围内
- 有效数据行数不足
- 所选列不含任何数值列
